## Design: Agent-Driven Landing Page Generation

### Problem

The landing page pipeline produces garbage output. Root cause: **the HTML is generated by a 1573-line Python template engine (`generator.py`) that produces generic centered-stack layouts**, not by the creative landing-page agent that has Phase 1-5 methodology for building bespoke experiences.

The `prompts.md` style guide (33 design systems, 3112 lines) is parsed and used only for LLM-powered design *selection* (fonts, colors, section names). The actual HTML rendering ignores the rich spec entirely — no signature moments, no asymmetric layouts, no experiential design, no special components.

**9 gaps identified by audit:**
1. `prompts.md` spec ignored during HTML generation (ROOT CAUSE)
2. Section types fall through to empty generic renderer
3. Research returns near-empty data for businesses without websites
4. Copy is generic/placeholder throughout
5. Landing-page agent (Phase 1-5) never invoked
6. Nav scroll effect hardcoded to white (breaks dark themes)
7. Canonical/OG URL points to fake domain
8. No signature moments, no asymmetric layouts, nothing unique
9. Two disconnected systems (API pipeline vs workflow pipeline)

### Approach

**Replace the hardcoded template engine with a Claude Code agent that receives the full `prompts.md` spec and generates HTML from scratch.** Keep the Python research steps (DDG scraping works fine for data gathering). Replace Steps 3+4 with a single agent invocation.

The core insight: Mev doesn't want us to pick a design template and fill in values — he wants the **landing-page-creator agent** to actually create unique, creative landing pages using the design systems in `prompts.md` as inspiration and style guides.

**New pipeline:**
```
POST /landing-pages/generate
  → _run_pipeline (background task)
    → Step 1: research_business()     — KEEP (Python DDG search + scrape)
    → Step 2: research_competitors()  — KEEP (Python DDG competitor scan)
    → Step 3: _generate_with_agent()  — NEW (Claude Code CLI agent)
```

Step 3 replaces both design synthesis and HTML generation with a single agent call that:
1. Reads the selected design spec from `prompts.md` (full text, not summary)
2. Gets all research + competitor data as context
3. Uses its Phase 1-5 methodology to generate complete HTML
4. Writes the HTML file directly to `/var/www/webassist/{id}/index.html`

---

### Key Decisions

1. **Agent invocation method: Claude Code CLI subprocess (`claude -p`)**: Use `claude --model claude-sonnet-4-6 -p "<prompt>" --max-turns 3 --allowedTools Write,Read,Bash` in a subprocess. This gives the agent fresh context (no accumulated pollution), access to filesystem (Write), and the landing-page.md agent methodology.

   - *Alternative rejected: workflow engine dispatch* — Adds unnecessary indirection. The workflow engine creates tasks that go through task_runner.sh and the full task lifecycle. For a single agent call that takes 30-60 seconds, a direct subprocess is simpler and faster.
   - *Alternative rejected: llm_chat() with better prompts* — This is what the current system does. The problem isn't the prompt — it's that `llm_chat()` returns text, not files. An agent can Write the HTML, verify it, and iterate.

2. **Design selection: Keep LLM-based selection from catalog, but pass the FULL spec to the agent**: The current `design_synthesizer()` picks a good design. The problem is downstream. We keep the selection step but instead of passing just fonts/colors, we pass the entire raw spec text from `prompts.md` to the agent.

   - *Alternative rejected: Let the agent pick the design too* — The agent would need to read all 3112 lines of prompts.md each time ($0.30+ in input tokens). Better to pre-select and pass only the relevant 200-400 line spec.

3. **Research enrichment: Use description field when URL scraping fails**: The `description` field from the user input is available but the pipeline ignores it downstream. Fix: merge description into research_data so the agent always has business context even without a working URL.

   - *Alternative rejected: Use a researcher agent for business research* — Over-engineering. The Python DDG scraping is fast (3-5s) and free. Agent research would cost $1-2 and take 60s+.

4. **No workflow engine integration**: The API pipeline runs as a FastAPI background task. This is correct for a product API that clients poll for status. The workflow engine is designed for internal orchestration. The landing page generation is a product feature, not an internal workflow.

   - *Alternative rejected: Wire API to workflow engine* — Workflow engine creates tasks visible in the OMS task queue, uses task_runner.sh with its full lifecycle (git preflight, QA review, etc.). This is overhead for a product API. Keep it as a background task.

5. **Agent output format: Direct file write**: The agent writes the HTML file directly. We verify the file exists after the agent completes. No need to parse HTML from stdout.

   - *Alternative rejected: Agent returns HTML as stdout, Python writes file* — Doubles the context window cost (HTML appears in both agent output and file). Direct write is cleaner.

6. **Budget and timeout**: $2 budget, 5 max turns, 120s timeout. The agent should generate the HTML in one shot with at most one revision pass.

#### Step 2: Build agent-driven HTML generation (core change)

**New file: `services/landing_page/agent_generator.py`** (~120 lines)

```python
"""Agent-driven HTML generation for landing pages.

Invokes Claude Code CLI with the landing-page-creator agent prompt,
passing the full prompts.md design spec + research data. The agent
writes HTML directly to /var/www/webassist/{id}/index.html.
"""

import asyncio
import json
import logging
import os
from pathlib import Path
from uuid import UUID

log = logging.getLogger("otto.agent_generator")

WEBASSIST_DIR = Path("/var/www/webassist")
BASE_URL = "https://webassist.otto.lk"
PROMPTS_PATH = Path("/mnt/media/prompts.md")
AGENT_PROMPT_PATH = Path("/home/web3relic/otto/.claude/agents/landing-page.md")


def _read_design_spec(design_id: str) -> str:
    """Extract the full spec for a specific design from prompts.md."""
    text = PROMPTS_PATH.read_text()
    # Find the design block
    import re
    pattern = rf'({re.escape(design_id)}\n.*?)(?=\nDESIGN \d|$)'
    match = re.search(pattern, text, re.DOTALL)
    if match:
        return match.group(1).strip()
    # Fallback: return first design
    first = re.search(r'(DESIGN 01\n.*?)(?=\nDESIGN \d|$)', text, re.DOTALL)
    return first.group(1).strip() if first else ""


def _build_agent_prompt(
    page_id: UUID,
    design_spec: str,
    design_decisions: dict,
    research_data: dict,
    competitor_data: dict,
) -> str:
    """Build the prompt for the Claude Code agent."""
    output_dir = WEBASSIST_DIR / str(page_id)
    output_path = output_dir / "index.html"
    preview_url = f"{BASE_URL}/{page_id}"
    
    business_name = research_data.get("business_name", "Business")
    
    return f"""You are building a landing page for {business_name}.

## YOUR TASK
Create a single self-contained HTML file at: {output_path}
Create the directory first: mkdir -p {output_dir}

## DESIGN SYSTEM (from prompts.md — follow this EXACTLY)
{design_spec}

## DESIGN DECISIONS (LLM-selected customization for this business)
{json.dumps(design_decisions, indent=2)}

## BUSINESS RESEARCH
{json.dumps(research_data, indent=2)}

## COMPETITOR ANALYSIS
{json.dumps(competitor_data, indent=2)}

## REQUIREMENTS
1. Single self-contained HTML file with ALL CSS inline in <style> tags and ALL JS inline in <script> tags
2. Google Fonts via <link> tag only — no other external dependencies
3. Follow the design spec PRECISELY — fonts, colors, layout structure, special components, animations
4. Mobile-first responsive: 375px → 768px → 1440px breakpoints
5. Implement the signature moment and special components from the design spec
6. NO centered vertical stack layouts — use asymmetric grids, offset content, editorial layouts as specified
7. IntersectionObserver for scroll-triggered animations
8. SEO meta tags: title, description, OG tags, Twitter Card
9. Accessibility: skip-to-content link, focus states, prefers-reduced-motion media query
10. Canonical URL: {preview_url}
11. OG URL: {preview_url}

## COPY GUIDELINES
- Headlines: punchy, specific, benefit-driven (NOT "Welcome to X")
- CTAs: action-oriented and specific (NOT "Learn More" or "Get Started" generics)
- If research data has real testimonials/reviews, use them. If not, create plausible ones that match the business
- Match the tone to the business: {research_data.get('tone_of_voice', 'professional')}

## ANTI-SLOP CHECKLIST (verify before writing)
- [ ] No font from banned list: Inter, Roboto, Arial, Open Sans, Lato, Montserrat, Poppins, Space Grotesk
- [ ] No centered-everything layout
- [ ] At least one grid-breaking element
- [ ] Signature moment implemented
- [ ] Color palette matches spec (not generic blue/white)
- [ ] Animations use spec's easing curve, not ease-in-out
- [ ] No placeholder images (use CSS shapes/gradients/patterns instead)
- [ ] No "Lorem ipsum" or "Coming soon" text

Write the complete HTML file now. After writing, verify the file exists and report the file size.
"""


async def generate_with_agent(
    page_id: UUID,
    design_decisions: dict,
    research_data: dict,
    competitor_data: dict,
    pool = None,
) -> dict:
    """Generate landing page HTML using Claude Code CLI agent.
    
    Returns dict with preview_url and html_path on success.
    Raises RuntimeError on failure.
    """
    # Get the full design spec
    design_id = design_decisions.get("selected_design_id", "DESIGN 01")
    design_spec = _read_design_spec(design_id)
    
    if not design_spec:
        log.warning("Could not extract design spec for %s, using full first design", design_id)
        design_spec = _read_design_spec("DESIGN 01")
    
    # Build the prompt
    prompt = _build_agent_prompt(
        page_id=page_id,
        design_spec=design_spec,
        design_decisions=design_decisions,
        research_data=research_data,
        competitor_data=competitor_data,
    )
    
    # Ensure output directory exists
    output_dir = WEBASSIST_DIR / str(page_id)
    output_dir.mkdir(parents=True, exist_ok=True)
    
    # Invoke Claude Code CLI
    cmd = [
        "claude",
        "--model", "claude-sonnet-4-6",
        "-p", prompt,
        "--max-turns", "5",
        "--allowedTools", "Write,Read,Bash",
        "--output-format", "text",
    ]
    
    log.info("[agent:%s] Starting Claude Code agent for HTML generation", page_id)
    
    try:
        proc = await asyncio.create_subprocess_exec(
            *cmd,
            stdout=asyncio.subprocess.PIPE,
            stderr=asyncio.subprocess.PIPE,
            cwd="/home/web3relic/otto",
            env={**os.environ, "CLAUDE_CODE_MAX_COST_DOLLARS": "2"},
        )
        
        stdout, stderr = await asyncio.wait_for(
            proc.communicate(),
            timeout=180,  # 3 minutes max
        )
        
        exit_code = proc.returncode
        log.info("[agent:%s] Agent exited with code %s", page_id, exit_code)
        
    except asyncio.TimeoutError:
        log.error("[agent:%s] Agent timed out after 180s", page_id)
        try:
            proc.kill()
        except Exception:
            pass
        raise RuntimeError("Agent timed out after 180 seconds")
    except Exception as exc:
        log.exception("[agent:%s] Agent invocation failed: %s", page_id, exc)
        raise RuntimeError(f"Agent invocation failed: {exc}")
    
    # Verify the HTML file was written
    html_path = output_dir / "index.html"
    if not html_path.exists():
        stderr_text = stderr.decode("utf-8", errors="replace")[:500] if stderr else ""
        log.error("[agent:%s] HTML file not created. stderr: %s", page_id, stderr_text)
        raise RuntimeError(f"Agent did not create HTML file. Exit code: {exit_code}")
    
    file_size = html_path.stat().st_size
    if file_size < 1024:
        log.warning("[agent:%s] HTML file suspiciously small: %d bytes", page_id, file_size)
        raise RuntimeError(f"Generated HTML too small ({file_size} bytes)")
    
    preview_url = f"{BASE_URL}/{page_id}"
    
    # Update DB if pool provided
    if pool:
        await pool.execute(
            """UPDATE landing_pages 
               SET html_path = $2, preview_url = $3, status = 'review', 
                   error_text = NULL, updated_at = now()
               WHERE id = $1""",
            page_id,
            str(html_path),
            preview_url,
        )
    
    log.info(
        "[agent:%s] HTML generated: %s (%d bytes) → %s",
        page_id, html_path, file_size, preview_url,
    )
    
    return {
        "html_path": str(html_path),
        "preview_url": preview_url,
        "file_size": file_size,
    }
```

### Data Flow (Fixed)

```
POST /landing-pages/generate
  │
  ├── INSERT landing_pages (status: pending)
  │
  └── _run_pipeline (FastAPI BackgroundTask)
       │
       Step 1: research_business()
       │  → DDG search + scrape → research_data JSON
       │  → description fallback ensures value_proposition populated
       │
       Step 2: research_competitors()
       │  → DDG competitor scan → competitor_data JSON
       │
       Step 3a: design_synthesizer()
       │  → LLM reads catalog summaries → picks DESIGN_XX
       │  → Returns design_decisions with colors/fonts/sections
       │
       Step 3b: generate_with_agent()    ← THIS IS THE FIX
       │  → Reads FULL spec for DESIGN_XX from prompts.md
       │  → Spawns Claude Code CLI with:
       │      - Full design spec (200-400 lines)
       │      - design_decisions (customized palette)
       │      - research_data + competitor_data
       │      - Anti-slop checklist
       │  → Agent generates HTML using Phase 1-5 methodology
       │  → Writes to /var/www/webassist/{id}/index.html
       │  → Verifies file exists and size > 1KB
       │  → Updates DB: status='review', preview_url set
       │
       └── If agent fails → fallback to template generator
```

### Risks

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Agent rate-limited (concurrent pipelines) | Medium | Medium | Fallback to template generator. Only 1 concurrent generation expected. |
| Agent generates broken HTML | Low | Medium | File size check (>1KB). Manual review at status='review'. Can regenerate. |
| Agent timeout (>180s) | Low | Medium | Kill process, mark as failed, allow retry. Most pages generate in 60-90s. |
| Claude Code CLI not available | Very Low | High | Verify `claude` in PATH at startup. This is Otto's primary compute engine. |
| Cost per page increases | Expected | Low | Agent call costs ~$0.50-1.50 vs $0.10 for template. Worth it for quality. Total per page: ~$1.50-3.00 (was $0.30-0.50). |
| prompts.md spec too long for context | Low | Medium | Individual design specs are 200-400 lines (~2-3K tokens). Well within limits. |
