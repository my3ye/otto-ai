## Strategy Overview

### Voice & Character
MY3YE is **The Eye That Sees What Must Be Built**. The X voice reflects this:
- Quiet authority — doesn't shout, doesn't hype
- Specific — names projects, cites real things, never vague
- Contrarian edge — calls out what's broken with existing systems
- Slightly cryptic in lore posts, direct in builder updates
- Web3-native but not degen — thoughtful signal in a noisy space
- No em-dashes, no "vibrant", no "crucial", no "game-changer"
- Uses "we" for community, "I" for the Eye's perspective

### Anti-Pattern Rules
Never:
- Generic AI/Web3 inspirational quotes
- "Excited to announce" energy
- Template-format threads (intro → 5 points → conclusion)
- Excessive hashtags (max 2 per post)
- Same format two days running
- External links every post (suppressed by algorithm)

### Posting Cadence
- **1-2 posts/day**, increasing to 2-3 in weeks 5-8
- **Primary window:** 13:30 UTC (9:30 AM EST, 7 PM IST)
- **Secondary window:** 22:00 UTC (6 PM EST, 3:30 AM IST)
- Variable timing — never exactly the same minute
- **No posting on Sunday** (give the account human-like rest pattern)

## Auto-Scheduler Integration

Posts are loaded into the Social Calendar with:
- `character: "my3ye"`
- `platforms: ["x"]`
- `status: "scheduled"` (or `"draft"` for review)
- `scheduled_at: "<ISO datetime UTC>"`

The X auto-scheduler (`x_scheduler.py`) runs every 15 minutes via systemd timer, picks up due posts, and broadcasts via the @maitrieye twikit session.

**Approval flow:**
1. Posts start as `draft`
2. Mev reviews and approves → status changes to `scheduled`
3. Auto-scheduler fires on schedule
4. Status updated to `posted`
