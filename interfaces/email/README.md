# Otto Email — Setup Guide

Otto's email capability uses Google Workspace for `otto@otto.lk` (SMTP + IMAP).

## Current DNS State (otto.lk)

| Record | Status |
|--------|--------|
| MX → smtp.google.com | ✅ Already set |
| SPF | ❌ Not set |
| DKIM | ❌ Not set (generated during Workspace setup) |
| DMARC | ❌ Not set |

## DNS Records Needed (Vercel DNS)

Add these in Vercel DNS for `otto.lk`:

### SPF (TXT record)
```
Name: otto.lk
Type: TXT
Value: v=spf1 include:_spf.google.com ~all
```

### DMARC (TXT record)
```
Name: _dmarc.otto.lk
Type: TXT
Value: v=DMARC1; p=quarantine; rua=mailto:otto@otto.lk; ruf=mailto:otto@otto.lk; fo=1
```

### DKIM (TXT record)
Generated by Google Workspace during setup. Will look like:
```
Name: google._domainkey.otto.lk
Type: TXT
Value: v=DKIM1; k=rsa; p=<key from Google Workspace Admin>
```

## Setup Steps for Mev

1. **Create Google Workspace account** for `otto.lk` domain
   - Go to workspace.google.com → Start free trial
   - Create user: `otto@otto.lk`

2. **Enable 2FA** on the otto@otto.lk Google account

3. **Generate App Password** (not regular password):
   - Google Account → Security → 2-Step Verification → App passwords
   - App: "Mail", Device: "Linux"
   - Copy the 16-character app password

4. **Add to `~/memory/.env`**:
   ```
   OTTO_EMAIL_ADDRESS=otto@otto.lk
   OTTO_EMAIL_PASSWORD=xxxx xxxx xxxx xxxx  (App Password)
   ```

5. **Add DNS records** (SPF, DMARC from above; DKIM from Workspace Admin)

6. **Enable IMAP** in Gmail settings:
   - Settings → See all settings → Forwarding and POP/IMAP → Enable IMAP

7. **Restart otto-memory service**:
   ```bash
   sudo systemctl restart otto-memory
   ```

8. **Verify**:
   ```bash
   curl http://localhost:8100/email/status
   # Should show: {"smtp": true, "imap": true, ...}
   ```

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | /email/status | Check SMTP + IMAP connectivity |
| POST | /email/send | Send an email |
| GET | /email/inbox | Fetch unread emails |
| POST | /email/inbox | Fetch emails (with options) |

### Send Example
```bash
curl -X POST http://localhost:8100/email/send \
  -H "Content-Type: application/json" \
  -d '{
    "to": "someone@example.com",
    "subject": "Hello from Otto",
    "body": "This is Otto checking in.",
    "from_name": "Otto"
  }'
```

### Inbox Example
```bash
curl http://localhost:8100/email/inbox?limit=10&unread_only=true
```

## Alternative: Resend (if Google Workspace not desired)

If Mev prefers not to use Google Workspace:
- Sign up at resend.com (3,000 emails/month free)
- Add domain `otto.lk`, follow their SPF/DKIM setup
- Use Resend SMTP: `smtp.resend.com:587` with API key as password
- For receiving: use Resend inbound webhooks → POST `/email/inbound`
- Update env:
  ```
  OTTO_SMTP_HOST=smtp.resend.com
  OTTO_EMAIL_ADDRESS=otto@otto.lk
  OTTO_EMAIL_PASSWORD=re_<api_key>
  ```

## Cost
