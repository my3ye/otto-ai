# Panik App — Comprehensive Roadmap
*Global public safety. One button. Human mesh network.*
*Last updated: 2026-03-16*

## Current Status
**CONCEPT** — panik.app domain. No code.

## Dependencies
- **Hard deps:** ONEON (mesh routing, identity for guardian network)
- **Soft deps:** Ottolabs (hardware mesh nodes for offline capability), Shakrah (mental health crisis protocol)
- **Blocks:** Nothing — can build in parallel with ONEON Phase 1 (web fallback while mesh is built)

---

## Phase 1 — MVP Safety Features (NOW → 45 days)
**Goal:** Core distress feature working on web. Guardian network bootstrapped.

### Milestones
1. **MVP scope locked** — One-button distress + guardian notification + location sharing
2. **Legal research done** — Emergency response liability in 2 pilot jurisdictions (Sri Lanka + 1 Western country)
3. **Guardian network model** — How guardians are recruited, verified, and rewarded
4. **Waitlist live** — panik.app waitlist with user vs guardian segmentation
5. **Web prototype** — Basic: press button → SMS/WhatsApp goes to pre-set guardians

### Success Criteria
- Legal research complete for 2 jurisdictions
- Guardian model designed and documented
- Waitlist ≥500 (users + guardians)
- Prototype demonstrates end-to-end distress signal

---

## Phase 2 — Mobile App Alpha (45 days → 4 months)
**Goal:** Real mobile app. First guardian network operational.

### Milestones
1. **iOS + Android app** — Native app: distress button, guardian management, location sharing
2. **ONEON identity** — Optional: prove identity via ONEON for higher trust tier
3. **Guardian verification** — Background check equivalent or community vouching system
4. **Mesh routing v0** — Initial: internet-based routing (ONEON layer). Mesh is Phase 3.
5. **1,000 active users with verified guardians**
6. **First real incidents handled** — Track safety outcomes, not just activations

### Success Criteria
- 1,000 active users with at least 1 verified guardian
- 99.9% distress signal delivery rate (no dropped signals)
- 0 false positives from panic button design (accidental activations handled gracefully)
- Response time ≤90 seconds for guardian notification

---

## Phase 3 — ONEON Mesh (4→12 months)
**Goal:** Panik works without internet. True offline safety net.

### Milestones
1. **ONEON mesh routing** — P2P signal routing via ONEON mesh (Bluetooth LE / local WiFi)
2. **Ottolabs hardware integration** — Otto Home and Otto Tower devices as mesh relay nodes
3. **Community responder network** — Trained first-responders within the ONEON community
4. **Emergency services integration** — API bridge to local emergency services (911/999/112) in pilot cities
5. **Mental health crisis protocol** — Panik-to-Shakrah handoff for non-physical emergencies
6. **100,000 active users**

### Success Criteria
- Distress signal works with 0 internet connectivity (full mesh mode)
- Emergency services integration in ≥2 cities
- 100,000 active users
- Measurable safety outcomes: response times, incident resolutions

---

## Phase 4 — Global Safety Infrastructure (12→24 months)
**Goal:** Panik is public safety infrastructure for communities worldwide. Especially conflict zones and underserved areas.

### Milestones
1. **Multi-language** — Core 20+ languages
2. **Conflict zone pilots** — Partnerships with NGOs operating in active conflict/displacement zones
3. **S0S Systems integration** — Panik data (anonymized) informs S0S refugee/displacement response
4. **Community watch networks** — Neighborhoods self-organize guardian coverage
5. **Hardware-backed safety** — Every Ottolabs device contributes to Panik mesh

### Success Criteria
- 1M+ active users
- Deployed in ≥1 active conflict zone or humanitarian crisis
- Partnership with ≥1 major humanitarian organization (UN, MSF, IRC)
- Measurably faster emergency response vs unassisted baselines

## The Panik Principle
Most safety apps require internet. Most safety apps require you to unlock your phone. Most safety apps don't work when you need them most.

Panik: one physical button (or gesture). Works offline. Your guardian network activates before you can think. The mesh gets there before the internet does.

## Revenue Model
- Free for users (safety cannot have a paywall)
- Premium guardian tiers: professional responders for hire
- Community partnerships: municipalities, universities, events (B2B)
- Hardware integration: Ottolabs devices include Panik mesh by default
