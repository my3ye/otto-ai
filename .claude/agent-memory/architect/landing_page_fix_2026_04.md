Landing page pipeline produces garbage output because HTML is generated by rigid Python templates, not by the creative landing-page agent.

**Why:** generator.py (1573 lines) uses hardcoded section renderers (_render_hero, _render_features, etc.) that produce generic centered-stack layouts. The prompts.md design spec (33 designs, 3112 lines) is parsed for design selection but completely ignored during HTML rendering. The landing-page.md agent (Phase 1-5 creative methodology) is never invoked.

**How to apply:** New file `services/landing_page/agent_generator.py` spawns Claude Code CLI (`claude -p`) with full design spec + research data. Design selection (design_synthesizer) kept as pre-filter. Template generator.py retained as fallback. Cost per page increases from ~$0.30 to ~$0.60-1.60 but output quality is the product. Full spec at ~/otto/docs/landing-page-fix-architecture-2026-04-06.md.
